The `ProjectResource` class represents a .NET project that can be run as part of your distributed application. It provides automatic service discovery, connection string injection, and seamless integration with the .NET ecosystem.

## Class Definition

```csharp
public class ProjectResource : Resource,
    IResourceWithEnvironment,
    IResourceWithArgs,
    IResourceWithServiceDiscovery,
    IResourceWithWaitSupport,
    IResourceWithProbes
{
    public ProjectResource(string name);
}
```

### Constructor

<ParamField path="name" type="string" required>
  Unique name for the project resource. Used for service discovery and references.
</ParamField>

## Adding Projects

### Basic Project

<CodeGroup>
```csharp Simple Project
builder.AddProject<Projects.Api>("api");
```

```csharp With Configuration
builder.AddProject<Projects.Api>("api")
       .WithHttpsEndpoint(port: 7001)
       .WithEnvironment("ASPNETCORE_ENVIRONMENT", "Production");
```

```csharp With Launch Profile
builder.AddProject<Projects.Api>("api", launchProfileName: "Production");
```
</CodeGroup>

<ParamField path="AddProject<TProject>(name, launchProfileName)" type="IResourceBuilder<ProjectResource>">
  Adds a .NET project to the application.
  
  **Type Parameters:**
  - `TProject` - Type from the generated `Projects` class (e.g., `Projects.Api`)
  
  **Parameters:**
  - `name` - Unique resource name
  - `launchProfileName` - Optional launch profile from launchSettings.json
  
  <Note>
    The `Projects` class is auto-generated by the Aspire SDK based on `<ProjectReference>` elements in your app host's .csproj file.
  </Note>
</ParamField>

## Service Discovery

Project resources automatically participate in service discovery. Other services can reference them to get automatically configured HTTP/HTTPS endpoints.

### Referencing Projects

<CodeGroup>
```csharp Backend-to-Database
var postgres = builder.AddPostgres("postgres");
var db = postgres.AddDatabase("mydb");

builder.AddProject<Projects.Api>("api")
       .WithReference(db);
// Injects: ConnectionStrings__mydb={connectionString}
```

```csharp Frontend-to-Backend
var backend = builder.AddProject<Projects.Api>("api")
                     .WithHttpsEndpoint();

builder.AddProject<Projects.Web>("frontend")
       .WithReference(backend);
// Injects: services__api__https__0={url}
```

```csharp Specific Endpoint
var api = builder.AddProject<Projects.Api>("api")
                 .WithHttpEndpoint(port: 8080, name: "admin");

builder.AddProject<Projects.Dashboard>("dashboard")
       .WithReference(api.GetEndpoint("admin"));
```
</CodeGroup>

### Service Discovery Environment Variables

When you reference a project with `.WithReference()`, these environment variables are automatically injected:

```bash Service Discovery Variables
# For HTTPS endpoint named "https":
services__api__https__0=https://localhost:7001

# For HTTP endpoint named "http":
services__api__http__0=http://localhost:5000

# Multiple endpoints of the same scheme:
services__api__https__0=https://localhost:7001
services__api__https__1=https://localhost:7002
```

These variables work with .NET's [Service Discovery](https://learn.microsoft.com/dotnet/core/extensions/http-client-service-discovery) to automatically configure `HttpClient` instances.

## Configuration

### Environment Variables

<CodeGroup>
```csharp ASPNETCORE_ Variables
builder.AddProject<Projects.Api>("api")
       .WithEnvironment("ASPNETCORE_ENVIRONMENT", "Production")
       .WithEnvironment("ASPNETCORE_URLS", "http://+:8080");
```

```csharp Logging Configuration
builder.AddProject<Projects.Api>("api")
       .WithEnvironment("Logging__LogLevel__Default", "Information")
       .WithEnvironment("Logging__LogLevel__Microsoft", "Warning");
```

```csharp Custom Application Settings
builder.AddProject<Projects.Api>("api")
       .WithEnvironment("MyApp__MaxRetries", "3")
       .WithEnvironment("MyApp__Timeout", "30");
```
</CodeGroup>

### Launch Profiles

Use launch profiles from launchSettings.json:

```json launchSettings.json
{
  "profiles": {
    "Development": {
      "commandName": "Project",
      "environmentVariables": {
        "ASPNETCORE_ENVIRONMENT": "Development"
      },
      "applicationUrl": "https://localhost:7001;http://localhost:5000"
    },
    "Production": {
      "commandName": "Project",
      "environmentVariables": {
        "ASPNETCORE_ENVIRONMENT": "Production"
      },
      "applicationUrl": "https://localhost:8443"
    }
  }
}
```

```csharp Using Launch Profiles
builder.AddProject<Projects.Api>("api", launchProfileName: "Production");
```

### Command-Line Arguments

<CodeGroup>
```csharp Static Arguments
builder.AddProject<Projects.Api>("api")
       .WithArgs("--urls", "http://+:8080");
```

```csharp Dynamic Arguments
var port = builder.AddParameter("api-port", "8080");

builder.AddProject<Projects.Api>("api")
       .WithArgs(context => 
       {
           context.Args.Add("--urls");
           context.Args.Add($"http://+:{port}");
       });
```
</CodeGroup>

## Endpoints

Project resources automatically expose HTTP and HTTPS endpoints from their Kestrel configuration.

### Adding Endpoints

<CodeGroup>
```csharp HTTP Only
builder.AddProject<Projects.Api>("api")
       .WithHttpEndpoint(port: 8080);
```

```csharp HTTPS Only
builder.AddProject<Projects.Api>("api")
       .WithHttpsEndpoint(port: 8443);
```

```csharp Both HTTP and HTTPS
builder.AddProject<Projects.Api>("api")
       .WithHttpEndpoint(port: 8080)
       .WithHttpsEndpoint(port: 8443);
```

```csharp External Access
builder.AddProject<Projects.Api>("api")
       .WithHttpsEndpoint(port: 8443, isExternal: true);
```
</CodeGroup>

### Endpoint References

<CodeGroup>
```csharp Get Endpoint Reference
var api = builder.AddProject<Projects.Api>("api")
                 .WithHttpsEndpoint();

var httpsEndpoint = api.GetEndpoint("https");

builder.AddProject<Projects.Frontend>("frontend")
       .WithEnvironment("API_URL", httpsEndpoint);
```

```csharp Multiple Endpoints
var api = builder.AddProject<Projects.Api>("api")
                 .WithHttpEndpoint(port: 8080, name: "http")
                 .WithEndpoint(port: 5001, name: "grpc", scheme: "http");

var httpEndpoint = api.GetEndpoint("http");
var grpcEndpoint = api.GetEndpoint("grpc");
```
</CodeGroup>

## Waiting for Dependencies

Ensure projects start in the correct order:

<CodeGroup>
```csharp Wait for Database
var postgres = builder.AddPostgres("postgres");
var db = postgres.AddDatabase("mydb");

builder.AddProject<Projects.Api>("api")
       .WithReference(db)
       .WaitFor(postgres);  // Wait for postgres to be ready
```

```csharp Wait for Migration
var postgres = builder.AddPostgres("postgres");
var db = postgres.AddDatabase("mydb");

var migration = builder.AddProject<Projects.DbMigration>("migration")
                       .WithReference(db);

builder.AddProject<Projects.Api>("api")
       .WithReference(db)
       .WaitForCompletion(migration);  // Wait for migration to complete
```
</CodeGroup>

## Replicas

Run multiple instances of a project:

<CodeGroup>
```csharp Multiple Replicas
builder.AddProject<Projects.Api>("api")
       .WithReplicas(3);
// Starts 3 instances: api-0, api-1, api-2
```

```csharp With Load Balancing
var api = builder.AddProject<Projects.Api>("api")
                 .WithReplicas(5)
                 .WithHttpsEndpoint();

builder.AddProject<Projects.Frontend>("frontend")
       .WithReference(api);
// Service discovery includes all replica endpoints
```
</CodeGroup>

<ParamField path="WithReplicas(count)" type="IResourceBuilder<ProjectResource>">
  Creates multiple instances of the project.
  
  **Parameters:**
  - `count` - Number of replicas (must be > 0)
  
  <Note>
    Each replica gets a unique name with an index suffix (e.g., `api-0`, `api-1`). Service discovery automatically includes all replica endpoints.
  </Note>
</ParamField>

## Publishing

When generating manifests for deployment, projects are published as container images.

```csharp Publishing Configuration
var api = builder.AddProject<Projects.Api>("api")
                 .WithHttpsEndpoint();

// During 'dotnet publish', this project will be:
// 1. Built as a container image
// 2. Included in the deployment manifest
// 3. Tagged with the registry from builder options
```

## Complete Examples

### Multi-Tier Application

```csharp Full Application
var builder = DistributedApplication.CreateBuilder(args);

// Database
var postgres = builder.AddPostgres("postgres")
                      .WithDataVolume();
var db = postgres.AddDatabase("appdb");

// Cache
var cache = builder.AddRedis("cache")
                   .WithDataVolume();

// Backend API
var apiKey = builder.AddParameter("api-key", secret: true);
var api = builder.AddProject<Projects.Api>("api")
                 .WithHttpsEndpoint(port: 7001)
                 .WithEnvironment("ExternalApi__ApiKey", apiKey)
                 .WithReference(db)
                 .WithReference(cache)
                 .WaitFor(postgres)
                 .WithReplicas(3);

// Worker Service
builder.AddProject<Projects.Worker>("worker")
       .WithReference(db)
       .WithReference(cache)
       .WaitFor(postgres);

// Frontend
builder.AddProject<Projects.Web>("frontend")
       .WithHttpsEndpoint(port: 7002)
       .WithReference(api)
       .WithExternalHttpEndpoints();

builder.Build().Run();
```

### Microservices with Service Discovery

```csharp Service Discovery Example
var builder = DistributedApplication.CreateBuilder(args);

// Shared services
var postgres = builder.AddPostgres("postgres");
var cache = builder.AddRedis("cache");

// Order Service
var orderDb = postgres.AddDatabase("orders");
var orderService = builder.AddProject<Projects.OrderService>("order-service")
                          .WithReference(orderDb)
                          .WithReference(cache)
                          .WithHttpsEndpoint();

// Inventory Service  
var inventoryDb = postgres.AddDatabase("inventory");
var inventoryService = builder.AddProject<Projects.InventoryService>("inventory-service")
                              .WithReference(inventoryDb)
                              .WithHttpsEndpoint();

// API Gateway (references both services)
builder.AddProject<Projects.ApiGateway>("gateway")
       .WithReference(orderService)
       .WithReference(inventoryService)
       .WithHttpsEndpoint(port: 7000, isExternal: true);

builder.Build().Run();
```
