---
title: Hosting Extension Methods
description: Extension methods for adding common resources to Aspire applications
---

Aspire provides a rich set of extension methods on `IDistributedApplicationBuilder` for adding resources to your application. These methods follow a consistent pattern and return `IResourceBuilder<T>` for fluent configuration.

## Pattern Overview

All resource extension methods follow this pattern:

```csharp
public static IResourceBuilder<TResource> Add{ResourceType}(
    this IDistributedApplicationBuilder builder,
    string name,
    // Resource-specific parameters
)
```

- Returns `IResourceBuilder<TResource>` for method chaining
- `name` parameter must be unique within the application
- Names are case-insensitive

## Container Resources

### AddContainer

Adds a generic Docker container.

<CodeGroup>
```csharp Latest Tag
builder.AddContainer("redis", "redis");
```

```csharp Specific Tag
builder.AddContainer("postgres", "postgres", "16-alpine");
```

```csharp With Configuration
builder.AddContainer("nginx", "nginx", "latest")
       .WithVolume("nginx-config", "/etc/nginx")
       .WithBindMount("./html", "/usr/share/nginx/html")
       .WithHttpEndpoint(port: 8080);
```
</CodeGroup>

<ParamField path="name" type="string" required>
  Unique name for the container resource.
</ParamField>

<ParamField path="image" type="string" required>
  Container image name (without tag).
</ParamField>

<ParamField path="tag" type="string" default="latest">
  Container image tag.
</ParamField>

### AddDockerfile

Builds a container from a Dockerfile.

<CodeGroup>
```csharp Basic Dockerfile
builder.AddDockerfile("myapp", "./src/MyApp");
```

```csharp Custom Dockerfile Path
builder.AddDockerfile("myapp", "./src/MyApp", "Dockerfile.production");
```

```csharp Multi-Stage Build
builder.AddDockerfile("myapp", "./src/MyApp", stage: "production");
```
</CodeGroup>

<ParamField path="name" type="string" required>
  Unique name for the container resource.
</ParamField>

<ParamField path="contextPath" type="string" required>
  Build context path (relative to app host or absolute).
</ParamField>

<ParamField path="dockerfilePath" type="string" default="Dockerfile">
  Path to Dockerfile relative to context path.
</ParamField>

<ParamField path="stage" type="string">
  Target stage in multi-stage Dockerfile.
</ParamField>

## .NET Project Resources

### AddProject

Adds a .NET project to the application.

<CodeGroup>
```csharp Basic Project
builder.AddProject<Projects.Api>("api");
```

```csharp With Configuration
builder.AddProject<Projects.Api>("api")
       .WithHttpsEndpoint(port: 7001)
       .WithEnvironment("ASPNETCORE_ENVIRONMENT", "Production")
       .WithReference(database);
```

```csharp With Launch Profile
builder.AddProject<Projects.Api>("api", launchProfileName: "Production");
```
</CodeGroup>

<ParamField path="TProject" type="Type" required>
  Type from the project's generated metadata (e.g., `Projects.Api`).
</ParamField>

<ParamField path="name" type="string" required>
  Unique name for the project resource.
</ParamField>

<ParamField path="launchProfileName" type="string">
  Name of the launch profile to use from launchSettings.json.
</ParamField>

<Note>
  The `Projects` class is generated by the Aspire SDK based on project references in your app host.
</Note>

## Executable Resources

### AddExecutable

Runs an arbitrary executable program.

<CodeGroup>
```csharp Simple Command
builder.AddExecutable("tool", "dotnet", "/path/to/project")
       .WithArgs("run");
```

```csharp Migration Script
builder.AddExecutable("migration", "dotnet", workingDirectory: "./db")
       .WithArgs("ef", "database", "update")
       .WithEnvironment("ConnectionString", database);
```

```csharp With Output
builder.AddExecutable("script", "bash", "/scripts")
       .WithArgs("setup.sh");
```
</CodeGroup>

<ParamField path="name" type="string" required>
  Unique name for the executable resource.
</ParamField>

<ParamField path="command" type="string" required>
  Command or path to executable.
</ParamField>

<ParamField path="workingDirectory" type="string" required>
  Working directory for the executable.
</ParamField>

### AddNpmApp

Runs a Node.js application using npm.

<CodeGroup>
```csharp React App
builder.AddNpmApp("frontend", "../ClientApp")
       .WithNpmCommand("start")
       .WithHttpEndpoint(port: 3000);
```

```csharp With Build
builder.AddNpmApp("frontend", "../ClientApp", "dev")
       .WithEnvironment("API_URL", api.GetEndpoint("https"));
```
</CodeGroup>

<ParamField path="name" type="string" required>
  Unique name for the npm app resource.
</ParamField>

<ParamField path="workingDirectory" type="string" required>
  Directory containing package.json.
</ParamField>

<ParamField path="scriptName" type="string" default="start">
  npm script to run from package.json.
</ParamField>

## Database Resources

<CodeGroup>

```csharp SQL Server Container
builder.AddProject<Projects.Api>("api")
       .WithReference(db);
```


</CodeGroup>

</ParamField>

## Parameter Resources

### AddParameter

Adds a configurable parameter that can be set at runtime.

<CodeGroup>
```csharp Simple Parameter
var apiKey = builder.AddParameter("api-key");

builder.AddProject<Projects.Api>("api")
       .WithEnvironment("API_KEY", apiKey);
```

```csharp Secret Parameter
var dbPassword = builder.AddParameter("db-password", secret: true);

var postgres = builder.AddPostgres("postgres", password: dbPassword);
```

```csharp With Default Value
var logLevel = builder.AddParameter("log-level", "Information");
```
</CodeGroup>

<ParamField path="name" type="string" required>
  Parameter name.
</ParamField>

<ParamField path="secret" type="bool" default={false}>
  Whether the parameter contains sensitive data (masked in UI).
</ParamField>

<ParamField path="default" type="string">
  Optional default value.
</ParamField>

<Note>
  Parameters can be set via:
  - Configuration (appsettings.json)
  - Environment variables
  - Command-line arguments
  - User secrets (for secret parameters)
</Note>

## External Services

### AddExternalService

References an external service not managed by Aspire.

<CodeGroup>
```csharp External API
var externalApi = builder.AddExternalService("external-api", 
    new Uri("https://api.example.com/"));

builder.AddProject<Projects.Api>("api")
       .WithReference(externalApi);
```

```csharp With Parameter
var apiUrl = builder.AddParameter("external-api-url");
var externalApi = builder.AddExternalService("external-api", apiUrl);
```
</CodeGroup>

<ParamField path="name" type="string" required>
  Resource name for the external service.
</ParamField>

<ParamField path="uri" type="Uri" required>
  Absolute URI of the external service (or parameter containing the URI).
</ParamField>

## Resource Naming

<Warning>
  Resource names must be unique within the application and are case-insensitive. Duplicate names throw `DistributedApplicationException`.
</Warning>

```csharp Name Validation
// This throws an exception:
builder.AddRedis("cache");
builder.AddPostgres("Cache");  // Case-insensitive conflict!

// Use unique names:
builder.AddRedis("redis-cache");
builder.AddPostgres("postgres-db");
```

## Method Chaining

All Add methods return `IResourceBuilder<T>` for fluent configuration:

```csharp Complete Configuration
builder.AddPostgres("postgres")
       .WithDataVolume("postgres-data")
       .WithEnvironment("POSTGRES_DB", "mydb")
       .WithLifetime(ContainerLifetime.Persistent)
       .AddDatabase("appdb");
```
